# 소제목 매핑 QA 검수 에이전트

PDF에서 추출한 원본 텍스트와, 소제목-내용 key-value로 파싱된 JSON 결과를 놓고
**"소제목별로 정확히 매핑됐는지"**를 검수하는 에이전트입니다.

## 폴더 구조

```
qa_agent/
├── section_spec.py       # 공통 소제목 데이터 형태 (code/title/group/aliases)
├── text_utils.py         # 정규화, 목차/페이지번호 제거, 유사도 계산 유틸
├── engine.py             # 핵심 검수 로직 (문서 유형과 무관하게 공통으로 동작)
├── registry.py           # document_type 문자열 -> 소제목 목록 매핑
├── configs/
│   ├── rfp.py            # 제안요청서 소제목 체계
│   ├── pep.py            # 사업수행계획서 소제목 체계
│   └── rpt.py            # 사업추진결과보고서 소제목 체계
└── run.py                # 로컬 테스트용 CLI
```

새 문서 유형이 생기면 `configs/`에 파일 하나 추가하고 `registry.py`에 등록만 하면
됩니다. `engine.py`(검수 로직)는 건드릴 필요 없습니다.

## 로컬 실행 방법 (백엔드/프론트 연결 전 테스트용)

```bash
python -m qa_agent.run --original original.txt --parsed parsed.json --doc-type rpt
```

- `--original`: PDF에서 추출한 원본 텍스트 파일 (.txt)
- `--parsed`: 소제목-내용 key-value로 파싱된 JSON 파일. 형식:
  ```json
  {
    "제1절 개요": "본문 내용...",
    "제2절 사업의 배경 및 목적": "본문 내용..."
  }
  ```
- `--doc-type`: `rfp` / `pep` / `rpt` 중 하나

실행하면 콘솔에 결과가 출력되고, `parsed파일명.review.json`으로 상세 리포트가
같이 저장됩니다.

## 검수 항목

1. **소제목 누락** (`missing_section`) — 기대 소제목이 파싱 결과에 없음
2. **빈 소제목** (`empty_section`) — 소제목은 있는데 내용이 비어 있음
3. **미등록 소제목** (`unrecognized_section`) — 파싱 결과에 기준 목록에 없는 소제목이 있음 (신규 항목이거나 분리 오류 가능성)
4. **순서 검수** (`section_order_mismatch`) — 원본과 파싱 결과의 소제목 순서가 다름
5. **섹션별 내용 매핑 검수** — 원본에서 그 소제목의 정확한 구간을 잘라내서 파싱 결과와 대조
   - `section_content_mismatch`: 내용이 원본과 안 맞음 (유실/변형)
   - `content_misplaced`: 내용이 **다른 소제목 자리**에 잘못 들어감 (어느 소제목으로 잘못 들어갔는지까지 짚어줌)

## 판정 기준

전체 유사도 숫자가 아니라 **"어떤 종류의 이슈가 있는가"**로 판정합니다.
(원본 전체엔 목차/장 제목/소제목 텍스트가 포함되지만 파싱 결과에는 본문만
있어서, 섹션이 전부 완벽해도 전체 유사도가 구조적으로 낮게 나올 수 있기
때문에 전체 유사도는 참고 지표로만 남기고 판정에는 쓰지 않습니다.)

- **FAIL**: `missing_section` / `empty_section` / `section_content_mismatch` /
  `content_misplaced` 중 하나라도 있으면 → 반려 코멘트 노출, 자동 진행 불가
- **WARN**: 위 치명적 이슈는 없지만 `unrecognized_section` / `section_order_mismatch`가
  있으면 → 참고 코멘트 노출, 자동 진행은 가능
- **PASS**: 아무 이슈도 없으면 → 코멘트 없이 바로 다음 단계(본 평가 에이전트) 진행 가능

리포트의 `can_auto_proceed` 필드가 이 자동 진행 가능 여부를 그대로 나타냅니다.

## 알려진 한계 / 다음에 다듬을 부분

- **동일 문구 반복**: 소제목 텍스트가 문서 안에서 여러 번 등장하면(예: 짧고
  흔한 표현), 위치 탐색이 엉뚱한 곳을 잡을 수 있습니다. 지금은 "직전 소제목
  위치 이후부터 검색"으로 최소화하고 있지만 완벽하진 않습니다.
- **숫자/날짜/금액 등 중요 토큰 검수는 이번 버전에서 제외**했습니다 (요청하신
  대로 섹션 매핑/내용 완전성에 우선 집중). 필요해지면 별도 모듈로 다시
  추가하면 됩니다.
- **alias 목록은 최소한으로만 채워둔 상태**입니다. 실제 문서를 몇 개 돌려보면서
  오탐(alias 부족으로 missing_section 잘못 뜨는 경우)이 나오면 각 config
  파일의 `aliases` 리스트에 계속 추가해나가면 됩니다.
