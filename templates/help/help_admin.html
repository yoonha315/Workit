도움말

시스템 관리자가 알아야 할 계정·부서 관리 안내입니다.

메뉴별 사용 가이드

관리자 화면의 핵심 기능을 정리했습니다.

계정관리
계정 생성 버튼으로 새 사용자를 등록하면 초기 비밀번호가 자동 부여되고, 최초 로그인 시 변경이 강제됩니다.
목록의 행을 클릭하면 정보 수정 모달이 열립니다 — 이름·연락처·소속부서·관리자 권한을 변경할 수 있습니다.
같은 모달에서 비밀번호 초기화, 강제 로그아웃, 계정 비활성화를 즉시 실행할 수 있습니다.
목록의 잠금/잠금해제 버튼은 5회 연속 로그인 실패 등으로 잠긴 계정을 즉시 풀어줄 때 사용합니다.
부서 관리
부서는 사용자와 문서(계약/이행) 접근 범위를 나누는 기준 단위입니다. 서로 다른 부서의 사용자는 서로의 문서를 볼 수 없습니다.
오른쪽 부서 추가 폼으로 새 부서를 등록한 뒤, 계정 생성 시 소속부서로 선택할 수 있습니다.
목록의 행을 클릭하면 부서명·부서코드 수정, 사용 여부 토글, 부서 삭제가 가능합니다.
소속된 사용자가 있는 부서는 삭제할 수 없습니다 — 먼저 사용자를 다른 부서로 옮기거나 비활성화하세요.
접속기록
최근 1년간의 모든 계정 로그인 시도(성공/실패) 기록을 확인할 수 있습니다.
행을 클릭하면 IP 주소와 User-Agent 등 상세 정보를 모달로 확인할 수 있습니다.
짧은 시간 내 반복된 실패 기록이나 낯선 IP가 보이면 보안 점검이 필요할 수 있습니다.
마이페이지
내 정보(이름·이메일·전화번호·직급)를 조회하고 수정합니다.
계정 설정에서 비밀번호를 변경하거나 로그아웃합니다.

자주 묻는 질문

Q관리자는 왜 계약관리·이행관리 화면이 안 보이나요?
관리자 계정은 시스템 운영(계정·부서 관리)을 전담하며, 부서별 계약·이행 문서는 정책상 조회하지 않습니다. 해당 메뉴와 URL은 관리자 계정에서 접근이 차단되어 있습니다.
Q계정을 잠금 처리하는 것과 비활성화하는 것의 차이는 무엇인가요?
잠금은 로그인 실패 누적·비밀번호 만료 등 운영 정책에 따른 일시적 제한이며 해제하면 즉시 다시 사용할 수 있습니다. 비활성화는 퇴사 등으로 더 이상 시스템을 쓰지 않는 계정을 막을 때 사용하며, 즉시 강제 로그아웃되고 다시 활성화하기 전까지 로그인이 불가합니다.
Q비밀번호를 초기화하면 사용자에게 어떻게 안내해야 하나요?
초기화하면 시스템 기본 초기 비밀번호로 재설정되고, 해당 사용자는 다음 로그인 시 비밀번호 변경이 강제됩니다. 초기 비밀번호는 별도 채널(전화, 사내 메신저 등)로 직접 안내하세요 — 화면에는 표시되지 않습니다.
Q부서를 삭제할 수 없다고 나옵니다.
해당 부서에 소속된 사용자가 한 명이라도 있으면 데이터 보호를 위해 삭제가 차단됩니다. 먼저 계정 수정 화면에서 사용자들을 다른 부서로 옮기거나 비활성화한 뒤 다시 시도하세요. 당장 쓰지 않을 부서라면 삭제 대신 사용 여부를 꺼두는 것도 방법입니다.
